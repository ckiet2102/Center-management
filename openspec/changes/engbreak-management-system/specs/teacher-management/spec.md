## ADDED Requirements

### Requirement: List all teachers
The system SHALL provide GET `/api/teachers` endpoint returning all teacher records with fields: id, full_name, code_name, dob, gender, phone, email, university, p_c (certifications), experience, specialty, note, status, avatar.

#### Scenario: Retrieve all teachers
- **WHEN** authenticated user sends GET `/api/teachers`
- **THEN** system returns HTTP 200 with array of teacher objects

### Requirement: Create teacher
The system SHALL provide POST `/api/teachers` endpoint to create a new teacher. Required field: full_name. The MySQL trigger `generate_teacher_codename` SHALL automatically generate the code_name.

#### Scenario: Successful teacher creation
- **WHEN** user sends POST `/api/teachers` with `{ full_name: "Nguyễn Văn Dũng", phone: "0901234567", email: "dung@email.com", status: "thu_viec" }`
- **THEN** system creates the teacher record (code_name auto-generated by trigger) and returns HTTP 201

### Requirement: Update teacher
The system SHALL provide PUT `/api/teachers/:id` endpoint to update teacher information including status changes. Status values: 'dang_day', 'tam_nghi', 'nghi_phep', 'nghi_viec', 'thu_viec'.

#### Scenario: Update teacher status
- **WHEN** user sends PUT `/api/teachers/1` with `{ status: "nghi_phep" }`
- **THEN** system updates the teacher status and returns HTTP 200 (MySQL trigger `track_teacher_status_change` automatically logs the change)

### Requirement: Teacher list UI
The frontend SHALL display teachers in a card or table view showing: tên, mã giảng viên, trạng thái (badge màu), chuyên ngành, kinh nghiệm, và chứng chỉ.

#### Scenario: View teacher list
- **WHEN** user navigates to the Teachers page
- **THEN** system displays all teachers with status badges color-coded (đang dạy=green, tạm nghỉ=yellow, nghỉ phép=blue, nghỉ việc=red, thử việc=purple)
